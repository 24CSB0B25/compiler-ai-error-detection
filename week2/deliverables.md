# Week 2 Deliverables – Literature Survey

## Tools and Technologies Reviewed
- Lex / Flex
- YACC / Bison
- GCC Compiler
- ANTLR
- Static code analysis tools

## Research Findings
- Traditional compilers use rule-based error detection.
- Existing tools do not intelligently classify malicious lexical patterns.
- Error messages are often difficult for beginners to understand.

## Research Gap Identified
- Lack of AI-based security-aware lexical analysis.
- No early-stage detection of malicious code patterns.
- Limited use of machine learning in compiler front-end phases.
