# WEEK 1: Problem Definition & Background

## Project Title
AI-Powered Secure Lexical Analyzer

## Problem Statement
Traditional compilers use rule-based lexical analyzers that can only
tokenize input programs and detect basic errors. They are unable to
identify malicious or suspicious lexical patterns such as injection
keywords or unsafe constructs at an early stage. This may allow
security vulnerabilities to go undetected.

## Objectives
- To understand compiler phases with focus on lexical analysis
- To study limitations of traditional lexical analyzers
- To design a security-aware lexical analyzer
- To enable early detection of malicious lexical patterns

## Scope
- Focuses only on lexical analysis phase
- Uses static analysis
- Supports limited programming language constructs
- Detects suspicious keywords and patterns

## Motivation
Early detection of malicious code during compilation improves software
security and reduces vulnerabilities.
