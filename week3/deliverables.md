# Week 3 Deliverables – Requirement Analysis

## Functional Requirements
- The system shall tokenize input source code.
- The system shall detect lexical errors.
- The system shall identify suspicious or malicious lexical patterns.
- The system shall generate security warnings.
- The system shall produce a threat report.

## Non-Functional Requirements
- The system should have high detection accuracy.
- The system should minimize false positives.
- The system should work efficiently with low overhead.
- The system should be user-friendly and readable.

## Input
- Source code written in a limited programming language subset.

## Output
- Token stream
- Error messages
- Security warnings and threat report
